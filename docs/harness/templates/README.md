# Harness Templates

Use these templates as the canonical source for recurring workflow artifacts.

## Core Templates
- `plan.template.md`: feature planning and implementation handoff
- `change-report.template.md`: completion and verification report
- `e2e-report.template.md`: E2E execution and failure/fix trace

## Specialized Templates
- `bugfix-root-cause.template.md`: bug workflow with hypothesis/evidence-first
- `implement-approved-plan.template.md`: execution flow for approved plans

## Rule
- Layer docs should reference these templates instead of duplicating field lists.
